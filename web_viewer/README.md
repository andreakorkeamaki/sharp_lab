# sharp_lab web_viewer

This is a static 3D Gaussian Splat viewer for files already generated by SHARP or other 3DGS pipelines. It is intentionally separate from the Python package so you can iterate on the core processing code and the browser viewer independently.

## What it does

- Load local `.ply`, `.spz`, `.splat`, or `.ksplat` files in the browser
- Load remote splat URLs if the host allows CORS
- Use Spark as the renderer, based on the official static import-map setup
- Stay deployable as a plain static site on Vercel

## Run locally

From the repo root:

```bash
cd web_viewer
python3 -m http.server 4173
```

Then open:

```text
http://localhost:4173
```

## Load your SHARP output

You can either:

- click `Local File` and choose a `.ply`
- drag and drop a `.ply` into the viewer
- host the file somewhere and pass a URL

If you want to open a remote file directly:

```text
http://localhost:4173/?url=https://example.com/scene.ply
```

## Vercel

This folder can be deployed as a static site. The viewer does not run SHARP in the browser; it only displays splat files that were already generated elsewhere.

Typical deployment flow:

1. Keep SHARP inference local or on a separate backend
2. Export a `.ply` or another supported splat format
3. Host the viewer on Vercel
4. Host the splat file on Vercel Blob, S3, R2, or another static file host with CORS enabled

## Notes

- Spark supports `.ply`, `.spz`, `.splat`, and `.ksplat`
- Large `.ply` files may be slower to load than optimized formats
- Browser access to remote files depends on CORS headers from the file host
